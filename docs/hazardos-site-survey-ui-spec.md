## Technical Implementation Notes

### Offline Storage

- Use IndexedDB via `idb` library
- Store complete survey object locally
- Queue photos for upload (store as blobs)
- Sync queue: First-in-first-out when online
- Conflict resolution: Last-write-wins with timestamp

### Photo Handling

- Compress before storage (max 1200px wide, 80% quality)
- Store original + thumbnail
- EXIF extraction for timestamp, GPS
- Upload queue separate from survey data

### Form State

- Zustand store for survey data
- Auto-save to IndexedDB on every change (debounced 500ms)
- React Hook Form for validation
- Zod schemas match database schema

### GPS Capture

```typescript
interface SurveyLocation {
  latitude: number;
  longitude: number;
  accuracy: number;
  capturedAt: string;
}
```

Capture on survey start, allow manual address entry override.

### Survey Status Flow

```

```

Field tech only sees `draft` status surveys they created.

## Accessibility

- Touch targets minimum 48x48px
- Color contrast ratio 4.5:1 minimum
- All icons have aria-labels
- Form fields have visible labels (not placeholder-only)
- Error states announced to screen readers
- Support system font scaling
